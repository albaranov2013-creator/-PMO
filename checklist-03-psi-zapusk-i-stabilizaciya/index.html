Чек-лист 03. ПСИ, запуск и стабилизация

Дата подготовки: 02.07.2026

Когда использовать

  • Перед стартом ПСИ.
  • На review результатов ПСИ.
  • Перед go / no-go.
  • Перед запуском банка.
  • На разборе хвостов и в начале стабилизации.
  • Перед передачей в сопровождение по клиентской стороне.

Критерий прохождения

  • Поздние фазы не идут вслепую.
  • Клиентская сторона подтверждена не только формально.
  • По хвостам и долгам внедрения есть ясное решение.

Блок 1. Готовность к ПСИ

ПСИ
  • Подтверждены клиентские участники приемки.
  • Подтверждены доступы, среда и организационная готовность клиента.
  • Подтвержден перечень модулей и сценариев приемки.
  • Понятно, как фиксируются замечания и кто их подтверждает.
  • Проведен синк аккаунт + PM перед стартом ПСИ.

Блок 2. Разбор результатов ПСИ

Результаты
  • Все замечания ПСИ собраны в явный перечень.
  • Замечания отделены от новых требований клиента.
  • По новым требованиям есть решение: входит, не входит, продается отдельно, переносится.
  • Понятно, что блокирует запуск, а что нет.
  • Клиентская позиция по критичным пунктам подтверждена.

Блок 3. Готовность к запуску

Go / No-Go
  • Есть подтвержденная клиентская готовность к запуску.
  • Есть подтвержденный состав модулей и опций запуска.
  • Есть подтвержденный режим работы в запусковое окно.
  • Проведен отдельный review go / no-go.
  • До клиента доведена единая позиция компании по рискам и ограничениям.

Блок 4. Хвосты после запуска

Post-launch
  • Сформирован список хвостов после запуска.
  • Хвосты разделены на: долги внедрения, новые доработки, сервисные вопросы, special case.
  • По каждому хвосту есть владелец и срок.
  • По каждому спорному хвосту есть клиентское согласование.
  • Аккаунт подключен к чувствительным постзапусковым вопросам.

Блок 5. Переход в стабилизацию и сопровождение

Stabilization
  • Понятно, какие вопросы остаются в проектном контуре.
  • Понятно, какие вопросы переходят в сопровождение.
  • Понятно, какие вопросы требуют отдельного коммерческого решения.
  • Клиент понимает модель работы после запуска.
  • Сводный контрольный реестр обновлен по поздним фазам.

Stop-факторы

Критично

Нельзя считать позднюю фазу управляемой, если:

  • Клиентская готовность к запуску не подтверждена.
  • Новые требования ПСИ не отделены от замечаний.
  • Решение go / no-go принимается без единой позиции компании.
  • Хвосты после запуска не разложены по категориям.
  • Клиентская сторона по долгам внедрения осталась неурегулированной.
