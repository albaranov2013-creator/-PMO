Чек-лист 02. Исполнение проекта и внешние встречи

Дата подготовки: 02.07.2026

Когда использовать

  • Перед регулярной внешней статусной встречей.
  • На еженедельном синке аккаунт + PM.
  • На review проблемного проекта.
  • На встрече по спорным требованиям клиента.
  • На клиентской эскалации.

Критерий прохождения

  • У компании есть единая позиция перед выходом к клиенту.
  • Спорные темы названы заранее.
  • После встречи можно зафиксировать конкретные решения, а не общие формулировки.

Блок 1. Синхронизация перед встречей

Подготовка
  • Синк аккаунт + PM реально проведен.
  • На синке подтвержден фактический статус проекта.
  • Согласовано, что именно говорим клиенту по срокам, объему и рискам.
  • Понятно, кто ведет встречу.
  • Понятно, кто фиксирует итоги и сроки.

Блок 2. Статус и проблемные зоны

Статус
  • Есть короткий понятный статус проекта.
  • Понятны текущая фаза и следующий обязательный рубеж.
  • Выделены красные зоны, которые нельзя замалчивать.
  • Понятны блокеры на стороне клиента.
  • Понятны блокеры на стороне delivery.

Блок 3. Спорные требования и scope

Scope
  • Все новые требования клиента собраны в явный перечень.
  • По каждому спорному пункту есть проектная оценка влияния.
  • По каждому спорному пункту есть коммерческая позиция.
  • Понятно, что входит в стоимость.
  • Понятно, что считается отдельно или переносится.

Блок 4. Качество клиентского контура

Контур клиента
  • Есть доступ к нужным ЛПР и рабочим ролям клиента.
  • Клиент не живет с параллельной картиной сроков и обязательств.
  • Значимые договоренности фиксируются в общем контуре.
  • Клиентские эскалации не остаются на PM без аккаунта.
  • Есть подтверждение, что следующий обязательный шаг клиента понятен и принят.

Блок 5. Выходы встречи

Результат
  • Есть протокол с решениями, сроками и ответственными.
  • Есть отдельная фиксация спорных вопросов.
  • Есть дата следующего обязательного события.
  • Есть решение, нужен ли дополнительный review / эскалация.
  • Данные обновлены в сводном контрольном реестре.

Stop-факторы

Критично

Встреча считается некачественной, если:

  • Она прошла без синка аккаунт + PM.
  • Стороны проговорили разные версии статуса.
  • Спорный scope остался без решения.
  • Клиент ушел со встречи с ожиданиями, которых delivery не подтверждает.
  • Итоги не зафиксированы письменно.
