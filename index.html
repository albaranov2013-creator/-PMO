Правила публикации

  • При каждой новой публикации в каталог добавляется новая строка с датой, slug, исходным файлом и итоговой ссылкой.
  • Markdown-реестр, JSON-реестр и web-каталог поддерживаются синхронно.
  • Каталог используется как общий навигационный вход для дальнейшей рассылки ссылок.
  • Если локальный файл уже связан с опубликованной страницей, после его изменения он считается кандидатом на обновление в GitHub Pages.
  • Перед любым внешним обновлением ассистент обязан отдельно спросить, нужно ли обновлять публикацию.
  • Без явного подтверждения пользователя внешнюю публикацию обновлять нельзя.
